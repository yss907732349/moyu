# 摸鱼隐者社区用户协议

版本：2026-06-08

本文档用于说明 `摸鱼隐者` 社区发布、评论、回复、举报和治理规则。实际上线前应结合主体信息、客服渠道和法务审核补齐正式条款。

## 社区身份与发布门槛

用户浏览社区、点赞、收藏和举报公开内容时，按既有登录、隐者档案和治理状态边界处理。用户发帖、评论或回复前，必须先同意当前版本隐私政策和社区用户协议，并完成微信手机号验证。

手机号验证只作为发布门槛和后台合规状态，不作为公开身份资产展示。社区公开页面不会展示完整手机号、手机号尾号或“已验证手机号”徽章。

若用户未登录、未创建隐者档案、未同意当前协议、未完成手机号验证，或账号处于限制、禁言、封禁状态，系统可以拒绝发帖、评论或回复，并给出对应提示。服务端拒绝后，小程序不得伪造提交成功或创建待审核内容。

## 内容发布与审核

用户应对自己发布的帖子、评论、回复和图片负责，不得发布违法违规、骚扰辱骂、色情低俗、广告引流、谣言误导、泄露隐私、侵犯他人权益或破坏社区秩序的内容。

社区内容可能经过本地规则、微信内容安全、AI 辅助审核和人工复核。低风险内容可以自动公开；需要复核的内容会进入待审核状态；明确违规内容可能被驳回、隐藏或移除。

待审核内容可以对作者本人显示“仅自己可见”，不代表内容已经公开。审核通过后其他用户可见；审核驳回、隐藏或移除后，公开侧不可见。

## IP 属地展示

用户成功发帖、评论或回复时，系统会由服务端根据可信请求来源生成发布时 IP 属地快照。帖子详情、评论、回复和公开个人主页会展示降精度 `IP属地`，例如省、自治区、直辖市、境外国家或地区，以及 `未知`。

用户不能关闭、隐藏或自定义 IP 属地展示。社区不会公开完整 IP、市、区县、街道、IP 来源 header 或解析原始结果。历史无属地快照内容不会被当前请求补造真实属地。

## 举报与治理

用户可以举报公开帖子、评论和回复，并选择结构化举报原因。系统会对重复举报、自己举报自己、非公开内容举报和频率进行限制。

后台可以根据内容安全结果、举报情况、历史有效举报风险和人工判断，对内容执行保留、隐藏、移除、标记误报等处理；也可以对用户执行限制、禁言、封禁或解除治理。

后台治理不依赖完整手机号或明文 IP。常规治理只使用内容 ID、作者用户 ID、手机号验证布尔状态、协议版本、降精度 IP 属地、举报原因和审核摘要。后台响应、实时待办和运营事件不得暴露完整手机号、手机号尾号、微信 openid、unionid、sessionKey、登录 token 或明文 IP。

## 异议与申诉

如果用户认为内容审核、举报处理、隐藏、移除、禁言或封禁结果存在错误，可以通过小程序内反馈入口或客服渠道提交说明。第一版不提供完整申诉中心、实名核身、身份证、人脸识别或手机号尾号排查能力；后续新增相关能力前，应先更新协议、隐私政策和实现规格。
