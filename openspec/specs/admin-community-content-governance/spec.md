## Requirements

### Requirement: 后台社区帖子总览

系统 SHALL 提供后台社区帖子总览，使运营人员可以查看所有用户发布的帖子，并默认按创建时间倒序排列。

#### Scenario: 读取所有帖子总览

- **WHEN** 携带有效 `x-admin-token` 的后台请求读取社区帖子总览
- **THEN** API SHALL 返回所有状态的社区帖子列表
- **AND** 列表 SHALL 默认按 `createdAt` 倒序排列
- **AND** 每个帖子项 SHALL 包含帖子 ID、标题、正文摘要、作者公开快照、作者用户 ID、分区、状态、创建时间、更新时间、风险摘要和互动统计摘要

#### Scenario: 按状态和分区筛选帖子

- **WHEN** 后台用户按帖子状态或分区筛选社区帖子总览
- **THEN** API SHALL 只返回符合筛选条件的帖子
- **AND** API SHALL 支持 `pending`、`approved`、`rejected`、`hidden` 和移除语义对应状态或标记的筛选

#### Scenario: 分页读取帖子总览

- **WHEN** 后台请求社区帖子总览并携带分页或游标参数
- **THEN** API SHALL 返回对应页的帖子列表
- **AND** API SHALL 限制单次返回数量，避免一次性返回全部历史内容

### Requirement: 后台社区关键词搜索

系统 SHALL 支持后台按关键词搜索社区帖子，并提供基础定位能力。

#### Scenario: 按标题和正文搜索帖子

- **WHEN** 后台用户输入合法关键词搜索社区帖子
- **THEN** API SHALL 返回标题或正文命中关键词的帖子
- **AND** 结果 SHALL 默认按创建时间倒序排列

#### Scenario: 按 ID 和作者定位帖子

- **WHEN** 后台用户输入帖子 ID、作者用户 ID 或作者昵称关键词
- **THEN** API SHALL 返回匹配帖子 ID、作者用户 ID 或作者公开昵称的帖子
- **AND** API SHALL NOT 要求运营人员只能通过公开列表逐页查找目标内容

#### Scenario: 搜索输入受限

- **WHEN** 后台用户提交空关键词、过短关键词或超过长度上限的关键词
- **THEN** API SHALL 拒绝搜索或按总览筛选规则处理
- **AND** API SHALL 返回后台可理解的输入错误提示

### Requirement: 后台社区帖子治理详情

系统 SHALL 提供后台帖子治理详情，用于查看帖子全文、评论、回复、举报和治理上下文。

#### Scenario: 查看帖子治理详情

- **WHEN** 携带有效 `x-admin-token` 的后台请求读取帖子治理详情
- **THEN** API SHALL 返回帖子全文、作者公开快照、作者用户 ID、分区、状态、创建时间、更新时间、审核摘要、低成本审核摘要、举报摘要和治理操作入口
- **AND** API SHALL 返回该帖子下的评论和一层回复上下文

#### Scenario: 查看评论和回复上下文

- **WHEN** 后台用户在帖子治理详情查看评论区
- **THEN** API SHALL 返回该帖子下所有状态的评论和一层回复
- **AND** 每条评论和回复 SHALL 包含内容、作者公开快照、作者用户 ID、状态、创建时间、风险摘要和可用治理操作

#### Scenario: 治理详情过滤敏感字段

- **WHEN** 后台读取帖子治理详情
- **THEN** 响应 SHALL NOT 包含薪资、工作档案、上班时间、隐藏模式、生存账单、消费统计、CPS 平台来源、微信 openid、微信 unionid、微信 sessionKey、用户登录态、明文 IP、AI 原始 prompt、供应商完整原始响应或供应商密钥

### Requirement: 社区举报案件治理详情

后台社区治理详情 SHALL 展示与当前帖子相关的举报案件、举报明细和处理历史，使运营人员可以在同一上下文中判断帖子、评论、回复和作者治理动作。

#### Scenario: 查看帖子举报聚合

- **WHEN** 后台用户打开帖子治理详情
- **THEN** API SHALL 返回该帖子及其评论、回复关联的举报聚合摘要
- **AND** 摘要 SHALL 包含待处理数量、已处理数量、举报原因分布、最高优先级、首次举报时间和最近举报时间

#### Scenario: 查看举报明细

- **WHEN** 后台用户在治理详情中展开举报明细
- **THEN** API SHALL 返回举报原因分类、补充说明、举报目标类型、举报目标 ID、举报状态、创建时间、处理时间和处理结果
- **AND** API SHALL NOT 返回举报人微信身份、应用登录态、工作档案、薪资、生存账单或其他与举报判断无关的敏感字段

#### Scenario: 定位评论或回复举报

- **WHEN** 举报目标是评论或回复
- **THEN** 后台治理详情 SHALL 在评论区上下文中标记对应目标
- **AND** 后台 SHALL 支持从举报明细跳转或滚动定位到被举报评论或回复

### Requirement: 举报与作者治理联动

后台 SHALL 允许运营人员基于举报审核结果对内容作者执行限制、禁言、封禁或解除限制等治理动作，并 SHALL 将举报处理和作者治理分别记录在审计历史中。

#### Scenario: 举报成立后限制作者

- **WHEN** 后台用户处理举报为隐藏或移除并同步限制作者
- **THEN** 系统 SHALL 更新被举报作者的治理状态
- **AND** 系统 SHALL 分别记录举报处理审计和作者治理审计
- **AND** 后续社区提交审核 SHALL 能识别该作者的治理限制状态

#### Scenario: 举报不成立不影响作者治理

- **WHEN** 后台用户处理举报为保留内容或误报
- **THEN** 系统 SHALL NOT 因该举报自动限制、禁言或封禁被举报作者
- **AND** 系统 SHALL NOT 将该举报计入被举报作者的有效举报历史

#### Scenario: 治理详情过滤敏感字段

- **WHEN** 后台读取包含举报信息的社区治理详情
- **THEN** 响应 SHALL NOT 包含薪资、工作档案、上班时间、隐藏模式、生存账单、消费统计、CPS 平台来源、微信 openid、微信 unionid、微信 sessionKey、用户登录态、明文 IP、AI 原始 prompt、供应商完整原始响应或供应商密钥

### Requirement: 后台内容治理操作

后台 SHALL 支持对帖子、评论和回复执行隐藏、移除和查看上下文等治理操作，并 SHALL 保留治理记录。

#### Scenario: 隐藏已公开帖子

- **WHEN** 后台用户对已公开帖子执行隐藏操作并提交原因
- **THEN** 系统 SHALL 将该帖子变为公开侧不可见
- **AND** 系统 SHALL 记录治理操作、原因、备注、操作人和操作时间

#### Scenario: 移除帖子

- **WHEN** 后台用户对帖子执行移除操作并提交原因
- **THEN** 系统 SHALL 将该帖子变为公开侧不可见
- **AND** 系统 SHALL 保留帖子内容和治理审计记录
- **AND** 系统 SHALL NOT 物理删除帖子、评论、回复、举报或通知记录

#### Scenario: 管理评论和回复

- **WHEN** 后台用户对评论或回复执行隐藏或移除操作并提交原因
- **THEN** 系统 SHALL 将对应评论或回复从公开评论区移除
- **AND** 系统 SHALL 保留原文、状态变化和治理审计记录

#### Scenario: 非法治理操作被拒绝

- **WHEN** 后台用户对不存在的帖子、评论、回复或不支持的状态执行治理操作
- **THEN** API SHALL 拒绝该操作
- **AND** API SHALL 返回后台可理解的错误提示

### Requirement: 后台作者治理与封禁

后台 SHALL 支持对社区作者执行禁言、封禁、限制和解除限制等治理操作，第一版 SHALL 以 `userId` 为封禁主键。

#### Scenario: 封禁帖子作者

- **WHEN** 后台用户从帖子治理详情对作者执行封禁并提交期限、原因和备注
- **THEN** 系统 SHALL 创建或更新该作者的社区治理状态
- **AND** 系统 SHALL 记录封禁开始时间、结束时间、原因、备注、操作人和操作时间

#### Scenario: 禁言帖子作者

- **WHEN** 后台用户对作者执行禁言并提交期限和原因
- **THEN** 系统 SHALL 将该作者限制为不可发帖、评论、回复或上传社区图片
- **AND** 系统 SHALL 保留用户浏览公开社区内容的能力，除非后续规格另行定义

#### Scenario: 解除用户限制

- **WHEN** 后台用户对被禁言、限制或封禁用户执行解除限制操作
- **THEN** 系统 SHALL 恢复该用户的社区写操作资格
- **AND** 系统 SHALL 记录解除操作、原因、操作人和操作时间

#### Scenario: 不使用明文 IP 作为第一版封禁依据

- **WHEN** 后台执行用户封禁
- **THEN** 系统 SHALL 以 `userId` 作为第一版封禁主键
- **AND** 系统 SHALL NOT 要求保存或展示明文 IP 才能完成封禁

### Requirement: 社区治理操作审计

系统 SHALL 记录后台社区治理操作，支撑复查、申诉和后续风控。

#### Scenario: 记录内容治理操作

- **WHEN** 后台用户隐藏或移除帖子、评论或回复
- **THEN** 系统 SHALL 记录目标类型、目标 ID、目标作者 ID、操作类型、旧状态、新状态、原因、备注、操作人和创建时间

#### Scenario: 记录用户治理操作

- **WHEN** 后台用户禁言、限制、封禁或解除限制社区用户
- **THEN** 系统 SHALL 记录目标用户 ID、操作类型、治理状态、期限、原因、备注、操作人和创建时间

#### Scenario: 读取治理历史

- **WHEN** 后台用户查看帖子详情或作者治理详情
- **THEN** API SHALL 返回与该帖子、评论、回复或用户相关的治理操作历史
- **AND** 治理历史 SHALL NOT 暴露后台令牌、用户登录态、微信身份或明文 IP

### Requirement: 后台社区治理验证

项目 SHALL 提供后台社区治理验证脚本，覆盖总览、搜索、详情、治理操作、封禁拦截和敏感字段边界。

#### Scenario: 运行后台社区治理验证

- **WHEN** 开发者运行后台社区治理验证命令
- **THEN** 验证 SHALL 检查社区帖子总览、关键词搜索、帖子治理详情、评论/回复管理、用户封禁和解封入口
- **AND** 验证 SHALL 检查后台请求携带 `x-admin-token`
- **AND** 验证 SHALL 检查治理响应不包含敏感字段黑名单

### Requirement: 社区治理实时待办一致性

后台社区治理 SHALL 与统一审核队列共享实时待办提示、筛选保留、操作反馈和状态变化处理语义。

#### Scenario: 从新待办进入社区治理

- **WHEN** 后台用户收到社区帖子、评论、回复或举报相关的新待办提示并进入社区治理
- **THEN** 社区治理列表 SHALL 保留或应用对应来源、状态、作者或风险筛选上下文
- **AND** 社区治理详情 SHALL 继续通过后台治理详情接口读取事实源

#### Scenario: 社区治理操作成功

- **WHEN** 后台用户在社区治理详情中隐藏、移除、限制作者、禁言、封禁或解除限制
- **THEN** 后台 SHALL 展示操作成功反馈
- **AND** 列表项和详情 SHALL 局部更新为最新状态或提示用户刷新

#### Scenario: 社区治理操作状态已变化

- **WHEN** 后台用户处理的帖子、评论、回复、举报或作者治理状态已经变化
- **THEN** 后台 SHALL 展示状态已变化或重复处理提示
- **AND** 后台 SHALL NOT 将该结果表达为操作成功但实际未变更

### Requirement: 社区治理状态和敏感字段边界

后台社区治理 SHALL 在实时提示、列表、详情和治理历史中保持一致的状态表达和敏感字段过滤。

#### Scenario: 社区治理列表状态扫读

- **WHEN** 后台用户查看社区治理总览
- **THEN** 列表 SHALL 清晰展示帖子状态、作者治理状态、风险摘要、举报摘要、创建时间和最近更新时间
- **AND** 列表 SHALL 支持后台用户快速区分待审、已通过、已驳回、已隐藏和已移除内容

#### Scenario: 社区治理详情状态扫读

- **WHEN** 后台用户查看社区治理详情
- **THEN** 详情 SHALL 清晰展示帖子、评论、回复、举报和作者治理的当前状态
- **AND** 详情 SHALL 只展示当前状态允许执行的治理操作

#### Scenario: 社区治理实时提示不暴露敏感字段

- **WHEN** 社区治理相关事件通过实时待办或轮询摘要提示后台用户
- **THEN** 提示数据 SHALL NOT 包含薪资、工作档案、上班时间、隐藏模式、生存账单、消费统计、微信 openid、微信 unionid、微信 sessionKey、用户登录态、明文 IP、AI 原始 prompt、供应商完整原始响应或供应商密钥
- **AND** 作者信息 SHALL 使用公开作者快照和后台治理所需的最小用户 ID

### Requirement: 社区治理验证扩展

项目 SHALL 扩展后台社区治理验证，覆盖实时待办一致性、操作反馈、筛选保留和敏感字段边界。

#### Scenario: 运行后台社区治理验证

- **WHEN** 开发者运行后台社区治理验证命令
- **THEN** 验证 SHALL 检查社区治理入口和详情在操作后展示明确反馈
- **AND** 验证 SHALL 检查社区治理响应和实时提示不包含敏感字段黑名单
- **AND** 验证 SHALL 检查筛选条件在刷新、返回和状态变化提示后保持可用

### Requirement: 社区治理内容安全摘要

后台社区内容治理 SHALL 在帖子、评论、回复和作者治理详情中展示归一化内容安全摘要，并 SHALL 继续隐藏微信身份和供应商原始响应。

#### Scenario: 治理详情展示审核来源

- **WHEN** 后台用户查看社区帖子、评论或回复治理详情
- **THEN** 响应 SHALL 展示本地规则、微信文本审核、微信图片审核、mock 或人工复核的归一化来源
- **AND** 响应 SHALL 展示归一化风险标签、简短原因、用户风险原因和处理建议

#### Scenario: 作者治理详情展示风险历史摘要

- **WHEN** 后台用户查看某个作者的社区治理详情
- **THEN** 响应 SHALL 可以汇总该作者近期自动驳回、人工复核、微信无法确认和审核通过的内容数量
- **AND** 响应 SHALL NOT 暴露该作者的完整 openid、unionid、`sessionKey`、微信内容安全原始响应或普通用户私密数据

#### Scenario: 治理操作不覆盖内容安全事实

- **WHEN** 后台用户隐藏、移除、通过或驳回社区内容
- **THEN** 系统 SHALL 保留原始归一化内容安全摘要用于后续排查
- **AND** 系统 SHALL 记录后台治理操作与内容安全自动判定之间的区别

### Requirement: 后台社区身份与 IP 合规摘要

后台社区治理 SHALL 在帖子、评论、回复和作者治理详情中展示必要的身份与 IP 合规摘要，并 SHALL 继续过滤完整手机号、明文 IP 和微信身份字段。

#### Scenario: 后台查看帖子治理详情

- **WHEN** 后台用户读取帖子治理详情
- **THEN** API SHALL 可以返回作者手机号验证布尔状态、隐私同意版本、帖子发布时 IP 属地标签和 IP 解析状态
- **AND** API SHALL NOT 返回完整手机号、手机号尾号、明文 IP、微信 openid、微信 unionid、微信 sessionKey 或用户登录态

#### Scenario: 后台查看评论和回复上下文

- **WHEN** 后台用户在帖子治理详情查看评论和回复上下文
- **THEN** 每条评论和回复 SHALL 可以包含发布时 IP 属地标签和 IP 解析状态
- **AND** 每条评论和回复 SHALL NOT 包含明文 IP 或 IP 来源 header

#### Scenario: 后台查看作者治理摘要

- **WHEN** 后台用户查看作者治理摘要
- **THEN** API SHALL 可以返回作者是否完成手机号验证、最近公开发布 IP 属地和当前社区治理状态
- **AND** API SHALL NOT 返回完整手机号、手机号尾号、明文 IP 或微信身份字段

#### Scenario: 后台查看 IP 解析失败内容

- **WHEN** 社区内容的 IP 属地解析失败
- **THEN** 后台 SHALL 可以看到解析失败摘要或 `未知` 标签
- **AND** 后台 SHALL NOT 因解析失败看到明文 IP

### Requirement: 后台身份与 IP 敏感字段验证

后台社区治理验证 SHALL 覆盖身份与 IP 合规摘要，并检查敏感字段不会进入后台响应。

#### Scenario: 运行后台社区治理验证

- **WHEN** 开发者运行后台社区治理验证命令
- **THEN** 验证 SHALL 检查帖子、评论、回复和作者治理详情包含必要的脱敏合规摘要
- **AND** 验证 SHALL 检查响应不包含完整手机号、明文 IP、微信 openid、微信 unionid、微信 sessionKey 或用户登录态

#### Scenario: 后台不依赖手机号或明文 IP 执行治理

- **WHEN** 后台用户隐藏、移除、限制、禁言或封禁社区用户
- **THEN** 系统 SHALL 继续以内容 ID、作者用户 ID 和治理状态执行操作
- **AND** 系统 SHALL NOT 要求后台查看完整手机号或明文 IP 才能完成治理
