## Context

当前小程序主导航已收敛为 `首页 / 社区 / 我的` 三个 tab，社区页也已完成高密度论坛列表方向。本 change 从首页、“我的”页和补给铺的入口层级优化，升级为小程序用户端全量生产级 UI 重构，并将默认视觉恢复为暗黑像素忍者 RPG 风格。

现状中，首页同时承载工作价值主卡、工作设置条、功能卡片、日报摘要、三张倒计时卡和底部生存消耗横幅；隐藏模式仍表现为工作档案设置中的长期配置项和功能入口，不符合“临时遮挡敏感信息”的真实使用场景。“我的”页顶部身份、资源和签到已具备真实数据基础，但成就系统尚未定义完整玩法，继续保留空成就区域会削弱完成感；下半区左侧菜单和右侧阵营插画布局可保留，但需要补充真实可用入口和语义 icon。

视觉层面，当前仍存在较多 `px-*` 占位文本、临时 icon、占位插画和系统字体。目标风格应接近“深色夜幕背景 + 暗色 RPG 卡片 + 像素插画 + 原生组件叠加”的完成态，但本 change 不要求一次性交付全部正式插画。首屏 banner、社区 banner、个人中心插画卡、补给铺板块、空状态等关键区域先预留稳定插画位、比例、安全区和 asset key；正式插画由后续制作完成后替换。金额、倒计时、按钮、列表、tabs、帖子、表单等动态内容仍使用原生 UI 组件渲染。不是每一个板块都必须使用插画底图，工具型信息密集区域可以保持暗色面板结构。

## Goals / Non-Goals

**Goals:**

- 首页第一屏形成清晰的上班仪表盘：用户首先看到今日已摸金额、工作状态、距离下班秒级倒计时和今日进度条。
- 隐藏模式成为首页主金额旁的即时隐私开关，并联动首页主体 banner 低调态。
- 首页今日生存消耗以一张大卡展示三类金额，并通过 `查看详情` 进入生存账本。
- “我的”页移除未上线成就占位，补充工作设置、生存账本等真实可用入口，并为每个入口提供语义 icon。
- 补给铺保持三类上班生存补给结构，视觉上像工具页而不是电商首页。
- 全局视觉采用深色夜幕背景、暗色卡片、细描边、克制辉光和像素 RPG 点缀，降低卡片和按钮之间的层级冲突。
- 接入正式像素字体，并明确授权、文件位置、加载方式、使用范围和降级策略。
- 交付插画资产预留清单和 fallback 规则，为后续正式插画替换留出稳定位置。
- 建立插画层与组件层的使用规则：该用插画承载氛围的地方要有完成度，该用组件承载信息的地方保持清晰可扫读。

**Non-Goals:**

- 不新增成就系统、称号商店、皮肤商店、排行、漫画独立 tab 或复杂游戏化玩法。
- 不改动社区页业务结构；社区页继续保持帖子列表、分区、发布和消息优先，但纳入视觉资产、字体和组件统一。
- 不重构后台运营端美术体系；后台继续以运营效率优先，除非后续 change 单独定义后台 UI 重构。
- 不新增后端 API、数据库字段或 CPS 订单同步规则。
- 不改变工作价值计算、生存账单统计或补给点击归因的业务口径。

## Decisions

### 1. 首页主卡以“距离下班”替代解释型时间文案

首页继续调用现有工作价值计算结果和计时器刷新机制，但主卡文案从“约等于多少时间”“今日进度百分比”调整为用户更关心的 `距离下班 HH:mm:ss`。今日进度条保留为辅助视觉反馈，但不再额外展示百分比文字。

替代方案是保留当前已工作时长和进度百分比。该方案信息完整但价值偏解释型，且与进度条重复，因此不作为主展示。

### 2. 隐藏模式从设置项改为首页即时开关

隐藏模式的使用场景是临时遮挡敏感金额，因此入口应靠近主金额。首页在 `今日已摸金额` 旁展示小眼睛 icon，点击后切换本地隐藏状态，遮挡已摸金额、每秒收益等工资推导信息，并让主体 banner 切换到低调视觉状态。

工作档案设置页不再展示隐藏模式开关。为了兼容现有数据契约，可以保留已有字段和降级读取逻辑，但新主流程不再把隐藏模式表达为工作档案配置。

### 3. 首页今日消耗是一张摘要大卡，不再像签到或任务入口

首页底部今日生存消耗改为一张大卡，顶部为标题和右上角 `查看详情` 文字按钮，中间三列平分展示 `隐者食堂`、`下午续命`、`通勤` 及金额。整卡或右上按钮均可进入生存账本详情。

替代方案是继续使用横向工具横幅。该方案可复用当前结构，但三类金额不够清楚，也容易被误认为临时活动或签到卡。

### 4. “我的”页保留下半区排版，只丰富内容

下半区保持左侧入口列表与右侧阵营插画卡的布局，避免在本 change 中重做整体结构。左侧入口扩展为 `我的角色`、`工作设置`、`我的帖子`、`收藏帖子`、`论坛消息`、`生存账本`，每项使用语义 icon。右侧阵营插画卡可叠加当前阵营修行状态、今日状态或低调态文案。

替代方案是改为全宽菜单列表。该方案信息更稳定，但违背当前希望保留排版的产品决策，因此不采用。

### 5. 成就系统暂不作为可见能力上线

成就系统玩法尚未定义，不应继续占据“我的”页首屏区域或菜单入口。原成就区域改为本人状态摘要，展示工作档案摘要和今日生存消耗摘要等真实能力。

后续如果要恢复“我的成就”，应通过新 change 定义成就来源、达成条件、展示规则、奖励和验收。

### 6. 补给铺以工具语义成熟化

补给铺仍展示 `隐者食堂`、`下午续命`、`通勤补给` 三个板块，但页面结构应更像上班补给清单：板块标题、场景说明、少量精选补给项和明确行动按钮。页面不得引入电商首页常见元素，如泛商城分类、购物车、收货地址、价格墙、券墙或平台促销瀑布流。

### 7. 采用“插画场景 + 原生组件叠加”的主视觉方法

用户端整体风格以暗黑 UI 和插画为主导，但插画不是铺满所有模块的装饰模板。首页主 banner、社区顶部 banner、个人中心阵营插画卡、补给铺三类板块头图、日报/大陆新闻封面、账本空状态和首跑引导可以使用插画场景作为底层氛围；金额、倒计时、列表、帖子、表单、筛选、按钮和菜单入口继续使用原生组件叠加或独立暗色面板承载。

这种方法比“所有卡片都做插画底”更适合工具型小程序：关键场景有记忆点，密集信息仍清楚。实现时应为插画容器定义稳定比例、安全留白和遮罩层，避免文字压住角色脸部、道具或关键场景。

### 8. 正式字体分层使用

接入一套具备合法授权的正式像素字体。字体文件应放在小程序静态资源或统一字体目录，并在全局样式中通过 `@font-face` 或小程序支持方式加载。字体使用范围分层：

- 品牌标题、金额数字、等级、倒计时、badge、入口标题可使用像素字体。
- 长正文、帖子内容、表单说明、错误提示应优先使用系统字体或可读性更好的中文字体。
- 如果像素字体不完整覆盖中文字符，必须提供明确 fallback，不得出现乱码、方块字或字重跳变明显的混排。

字体资源必须记录来源和授权，不得提交来源不明或不可商用字体。

### 9. 暗黑主题与 RPG 氛围边界

默认页面背景使用深色夜幕、黑蓝或深紫黑，主 tab 默认背景回到暗色。卡片使用暗色浮层、细描边和克制辉光形成层次；文字以米白、灰蓝为主，强调色保留金色、紫色、蓝色、绿色、橙色等 RPG 状态点缀。

浅色元素只作为局部对比存在，例如吐槽气泡、纸张、卷轴、图标高光或插画道具。浅色不得重新成为整页背景。

### 10. 插画预留清单驱动替换

实现前应先建立用户端插画预留清单，至少覆盖：

- 首页：普通工作主 banner、隐藏模式 banner、日报入口图、补给入口图、倒计时 icon、今日消耗图标。
- 社区：社区顶部 banner、分区/筛选/发布/消息 icon、默认帖子缩略图、空列表和未登录状态图。
- 我的：四阵营头像、四阵营徽章、四阵营插画、菜单入口 icon、签到和资源 icon。
- 补给铺：三类补给板块插画、补给项默认封面、转链失败和空板块状态图。
- 内容页：日报、大陆新闻、账本、工作设置、登录/建档、空状态、错误态和加载态所需插画或 icon。

所有 `px-* 200×200`、`badge_placeholder`、纯文字资产占位和无语义圆点 icon 都应被稳定插画位、风格化 fallback 或合法的组件化 icon 替换。正式插画文件可以在后续制作完成后按相同 asset key 接入，不应要求再改页面结构。

## Risks / Trade-offs

- [Risk] 隐藏模式从工作档案设置页移除后，老用户已保存的 `hiddenMode` 字段可能仍存在 → 保留字段兼容读取，但新 UI 以首页本地开关为主。
- [Risk] 首页信息收口后，日报等内容入口曝光降低 → 保留核心快捷入口，日报仍可进入，但不再抢占工作状态主卡。
- [Risk] “我的”页入口增加后左侧列表可能过挤 → 固定最多 6 个主入口，每个入口使用短标题、语义 icon 和稳定行高。
- [Risk] 右侧阵营插画叠加状态文案可能遮挡图片主体 → 文案只使用短句和轻遮罩，插画核心内容保持可见。
- [Risk] 今日消耗为 0 时卡片显得空 → 三类金额明确展示 `¥0.00`，并保持 `查看详情` 入口解释订单同步后的自动归类。
- [Risk] 插画风格过重导致工具信息难扫读 → 仅在主视觉、banner、状态和入口卡使用插画，表单、列表和密集数据继续使用原生组件面板。
- [Risk] 正式像素字体影响中文可读性或包体积 → 限定使用范围，正文使用系统字体 fallback，并优先选择可商用、体积可控的字体文件。
- [Risk] 正式插画尚未完成导致页面空洞 → 本 change 先交付稳定插画容器、安全区和风格化 fallback，不展示开发占位文本，正式插画后续按 asset key 替换。
- [Risk] 暗色风格造成信息压抑或可读性下降 → 通过更克制的卡片层级、足够对比度、减少过强发光和控制字号密度保持工具可读性。

## Migration Plan

1. 建立用户端插画预留清单、命名规范和字体接入方案，确认字体授权与静态资源目录。
2. 更新默认 feature registry 和页面入口消费逻辑，使首页不再展示 `hide_mode` 独立快捷入口，“我的”页不展示成就入口或未来态入口。
3. 更新全局视觉 token、字体声明、暗色页面背景、暗色卡片、按钮、icon 容器和插画容器样式。
4. 更新首页模板和样式，重排主卡、快捷入口、倒计时和今日消耗卡，并接入插画预留位与风格化 fallback。
5. 更新工作档案设置页，移除隐藏模式 UI，保留保存契约兼容，并统一表单视觉。
6. 更新“我的”页，替换成就区域为本人状态摘要，在既有下半区布局中扩展入口与 icon，并接入阵营头像、徽章和插画。
7. 更新社区、日报、大陆新闻、生存账本、补给铺、WebView 回退页、登录/建档、空状态和错误态的插画预留位、fallback 与字体。
8. 扫描并清理用户端所有占位资产文本、临时 icon 和无语义圆点 icon。
9. 运行相关验证命令，重点覆盖小程序页面、feature registry、工作价值、账本摘要、补给铺、社区和内容页。

## Open Questions

- 首页补给入口标题最终使用 `隐者食堂` 还是 `补给铺`：推荐首页卡片标题偏场景化使用 `隐者食堂`，落地页仍叫 `补给铺`。
- 隐藏模式是否遮挡今日生存消耗金额：推荐第一版遮挡首页所有金额类敏感信息，包括今日消耗；生存账本详情页暂不强制遮挡。
- “我的”页本人状态摘要是否展示工作时段：推荐展示上班时段和发薪日，不展示月薪。
- 正式像素字体当前选型记录为 Fusion Pixel / OFL-1.1，后续仍需在正式字体文件完整入库前确认小程序包体成本、中文覆盖范围和多机型渲染表现。
- 正式插画接入批次：本 change 只要求预留位置和稳定 key，正式插画由后续制作完成后按预留 key 替换。
