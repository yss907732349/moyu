## Why

社区已经具备发帖、评论、回复、举报和作者公开快照，但用户只能在内容流里看到一次性作者信息，缺少进入作者公开身份、建立轻量关注关系和查看本人关注/粉丝的闭环。随着社区继续从内容发布走向弱社交，需要先补齐公开个人页和单向关注关系，同时保持隐私、治理和未开放成长资源的边界。

## What Changes

- 新增社区公开个人页：评论区、回复区和帖子作者头像或昵称可点击进入公开个人页。
- 公开个人页展示头像、昵称、阵营、等级、称号、粉丝数、关注数、总公开发帖数和公开帖子列表；若社区身份与 IP 属地能力已落地，则沿用最近一次公开发布行为的 `IP属地` 展示口径。
- 新增关注/取消关注能力，支持查看关注列表和粉丝列表；关注关系为单向关系，不引入好友确认。
- 小程序“我的”页顶部身份区域保持三项状态数据，展示关注数、粉丝数和连续签到天数；关注数和粉丝数可点击进入对应列表。
- 小程序普通用户界面暂时隐藏隐币和能量统计；相关字段可继续作为内部成长资料数据存在，后续若恢复展示需要另起 change 定义用途、获取来源、消耗场景和展示位置。
- 社区作者快照需要提供可点击公开个人页的稳定公开标识，不使用微信 `openid`、手机号、昵称或可直接枚举的内部身份字段作为公开路由凭据。
- 公开个人页、关注列表、粉丝列表和社区作者入口继续过滤手机号、真实姓名、完整 IP、微信身份字段、登录态、薪资、工作时间、工作档案、隐币、能量、生存账单、消费统计、CPS 来源、后台治理状态和内容安全内部风险信息。
- 暂不做私信、拉黑、动态流、关注流、推荐算法、排行榜、新增粉丝通知、好友关系、用户搜索、头像上传、隐币/能量玩法公开或关注关系后台运营面板。

## Capabilities

### New Capabilities

- `community-follow-profile-system`: 定义社区公开个人页、公开作者跳转标识、关注/取消关注、关注/粉丝列表、公开统计、公开帖子列表和敏感字段边界。

### Modified Capabilities

- `community-lite`: 社区帖子、评论和回复作者头像/昵称支持进入公开个人页，并保持公开内容、作者快照和审核状态过滤边界。
- `user-growth-profile`: “我的”页顶部状态数据从隐币/能量/连续签到调整为关注数/粉丝数/连续签到，并暂时隐藏隐币和能量普通 UI 展示。
- `visual-system`: 补充公开个人页、关注/粉丝列表和“我的”页三项状态数据的暗黑像素 RPG 布局要求。
- `mvp-beta-readiness`: 补充公开个人页、关注关系、关注/粉丝列表、敏感字段过滤和隐币/能量隐藏的内测验收。

## Impact

- 小程序：`apps/miniapp` 社区帖子详情页、评论/回复组件、公开个人页、关注列表/粉丝列表、“我的”页顶部身份区域、社区客户端服务和用户资料客户端服务。
- 后端 API：`apps/api` 公开个人页查询、关注/取消关注、关注列表、粉丝列表、公开帖子列表、本人关注状态、统计计数和敏感字段过滤。
- 数据模型：新增关注关系表或等价持久化结构，并为公开个人页提供稳定公开标识和计数聚合；计数可冗余但应以关系事实为准。
- 共享契约：`packages/shared` 公开个人页、关注关系、公开用户列表项、作者公开路由标识、关注状态和敏感字段黑名单相关类型。
- 文档与验证：`docs/` 内测检查清单和 `scripts/` 社区/个人页/用户成长资料相关验证脚本。
