## Overview

本变更以“真实状态优先”为原则稳定小程序首跑链路。小程序不维护独立的“首跑完成”事实标记，而是每次根据应用登录态、用户成长资料和工作档案状态推导当前用户应看到的下一步。

## State Model

首跑状态分为：

- `logged_out`: 本地没有有效应用登录态。
- `profile_missing`: 已登录，但远端明确没有用户成长资料，且本地没有合法用户成长资料快照可降级。
- `work_profile_missing`: 已有合法用户成长资料或快照，但远端工作档案未配置，且本地没有合法工作档案快照可降级。
- `ready`: 已有合法用户成长资料或快照，并已有合法工作档案快照或远端已配置工作档案。

状态推导不得被本地引导偏好、上次步骤记录或页面访问历史覆盖。

## API Configuration

小程序所有客户端服务通过 `apps/miniapp/src/services/api-config.ts` 读取同一个 API base URL。`VITE_API_BASE_URL` 存在时优先使用环境变量；未配置时使用项目默认开发地址。真机调试需要局域网地址时，应通过环境配置显式设置，不在单个服务里硬编码特殊地址。

## Page Flow

首页负责展示当前首跑状态对应的主要行动：

- 未登录：引导登录或进入“我的”页登录。
- 已登录未建档：引导创建隐者档案。
- 已建档未配置工作档案：引导配置薪资、上班时间和发薪日。
- 已完成：使用工作档案快照计算今日已摸金额、工作状态、倒计时和每秒收益。

个人页在创建隐者档案成功后保存合法用户成长资料快照，并提供配置工作档案的明确入口；如果工作档案已经存在，该入口不再展示为待办提示。

工作档案设置页保存成功后写入合法 `WorkProfileSnapshot`，并引导或返回首页。首页返回时优先读取最新本地快照并立即计算。

## Empty State Boundaries

日报、社区和生存账本需要区分真实空数据和接口错误：

- 真实 0 或无内容应展示空状态。
- API 错误应展示暂不可用或重试提示。
- 生产公开侧不得自动伪造日报、帖子或用户消费账单。
- 开发或演示种子只能由显式脚本或显式准备流程创建，并保持隐私和来源完整性边界。

## Verification

验收覆盖 OpenSpec 严格校验、格式、lint、类型检查、小程序 API 配置验证、用户成长资料、工作档案、日报、社区和生存账本验证。`build:miniapp` 曾在当前沙箱出现 `spawn EPERM`，后续已通过调整包构建串行化和在非沙箱环境复验处理。
