## Why

当前社区已经具备发帖、评论、回复、内容安全、举报和后台治理能力，但发布型论坛仍缺少真实身份认证、隐私同意留痕和 IP 属地公开展示这一层合规基础。随着社区继续往粉丝关注、个人主页和更完整社交关系推进，需要先把发布门槛、公开属地、后台脱敏治理和内测审核口径统一固化。

## What Changes

- 新增社区身份与 IP 属地合规层：用户发帖、评论和回复前必须先同意当前隐私政策/用户协议，再完成微信手机号验证。
- 明确手机号验证只作为发布门槛和后台合规状态，不作为公开身份资产展示；公开侧不展示手机号、手机号尾号或“已验证”徽章。
- 服务端在发帖、评论和回复时从可信请求来源计算客户端 IP，并生成发布时 IP 属地快照；不得信任小程序前端传入的 IP 或属地。
- 小程序个人主页展示用户最近一次公开发布行为的 IP 属地；帖子详情、评论和回复预留轻量展示位，提审包和生产环境默认展示，帖子列表默认不展示以保持列表密度。
- IP 属地公开展示范围限定为境内省/自治区/直辖市、境外国家或地区；不得显示完整 IP、市、区县、街道，也不得提供关闭 IP 属地开关。
- 后台社区治理展示隐私同意版本、手机号验证状态、IP 属地摘要和解析失败提示，但不得展示完整手机号、完整 IP、微信 openid、unionid 或登录态。
- 补充隐私政策、用户协议、内测检查清单和验证脚本，覆盖手机号验证、IP 属地、隐私同意版本、公开展示边界和后台敏感字段过滤。
- 暂不接入身份证号、人脸实名核身、真实姓名展示、完整手机号展示、完整 IP 展示、设备指纹、城市级属地或用户可关闭属地能力。

## Capabilities

### New Capabilities

- `community-identity-and-ip-attribution`: 定义社区发布身份门槛、隐私同意、手机号验证、IP 属地归因、公开展示范围和敏感字段边界。

### Modified Capabilities

- `community-lite`: 发帖、评论、回复接入手机号验证门槛和 IP 属地快照；社区详情、评论、回复和个人主页展示 IP 属地，帖子列表保持不展示。
- `user-growth-profile`: 扩展本人资料/公开主页所需的隐私同意、手机号验证状态和最近公开发布 IP 属地摘要，但不公开手机号验证状态。
- `admin-community-content-governance`: 后台社区治理详情补充脱敏身份状态、隐私同意版本、IP 属地摘要和解析失败提示，并继续过滤完整手机号和明文 IP。
- `admin-operations`: 统一审核队列、实时待办和后台摘要补充身份/IP 合规检查边界，保持敏感字段黑名单。
- `mvp-beta-readiness`: 更新内测配置、人工检查清单和总 smoke 验证，覆盖隐私同意、手机号验证、IP 属地展示和提审/生产显示开关。

## Impact

- 小程序：`apps/miniapp` 社区发帖页、帖子详情页、评论/回复组件、个人主页/后续关注主页、社区客户端服务、隐私同意弹窗和手机号验证入口。
- 后端 API：`apps/api` 微信手机号验证接口、本人身份状态接口、社区发帖/评论/回复写入口、可信客户端 IP 提取、IP 属地解析、后台治理响应和 Prisma 数据模型/迁移。
- 管理后台：`apps/admin` 社区治理详情、作者治理摘要、统一审核队列和敏感字段过滤。
- 共享契约：`packages/shared` 身份门槛错误码、隐私同意版本、手机号验证状态、IP 属地标签、社区公开响应和后台治理响应类型。
- 文档与验证：`docs/` 隐私政策/用户协议/内测清单，`scripts/verify-community-lite*`、`scripts/verify-admin-community-governance*`、`scripts/verify-mvp-beta-readiness*` 等验证脚本。
