## Why

当前小程序用户端以暗黑忍者 RPG 视觉作为默认外观，但真实使用场景主要发生在白天工位。默认暗色会增加办公环境中的暴露感，也会让“伪装上班工具”的核心场景不够自然。

本 change 将浅色“工位模式”提升为小程序用户端全局默认外观，同时保留暗黑 RPG “隐者模式”作为首页快捷切换的沉浸体验，确保用户无需进入设置即可在工位界面和隐者界面之间切换。

## What Changes

- 将小程序用户端默认视觉从暗黑 RPG 改为浅色工位模式。
- 保留暗黑忍者 RPG 视觉，并命名为“隐者模式”，由首页右上角快捷按钮主动进入。
- 首页右上角切换入口显示目标动作：工位模式下显示“隐者模式”，隐者模式下显示“返回工位”。
- 主题选择仅本地保存，重新打开小程序后保持上次选择。
- 小程序用户端所有已注册页面都必须完成工位模式组件级视觉适配，不再暂缓日报、大陆新闻、我的社区、消息、协议、漫画兼容页或 webview 外壳。
- 页面内所有可见 UI 单元和状态都必须适配，包括背景、卡片、列表、按钮、表单、chip、badge、头像框、图片框、插画遮罩、进度条、弹层、固定底栏、空状态、加载态、错误态、骨架屏、选中态、禁用态、按压态和加载中状态。
- `stealth-workbench/*` 继续作为办公表格视觉例外，不因隐者模式切换而强行改成暗黑 RPG，但必须与默认工位模式不冲突。
- 插画、头像、像素 icon 和用户/商品/社区图片先做可读性适配，不重新设计整套白版资产。
- 不修改用户登录、首跑、工作价值计算、隐藏模式、发帖、关注、审核、补给跳转、账本统计或其他业务逻辑。

## Capabilities

### New Capabilities

- 无。

### Modified Capabilities

- `visual-system`: 将小程序用户端默认视觉改为工位模式，新增工位模式/隐者模式切换、全页面全组件主题适配、主题本地持久化、运行时导航栏/tabBar 适配和摸鱼工作台视觉例外边界。

## Impact

- 影响 `apps/miniapp` 所有用户端页面、全局样式、页面内局部样式、共享视觉组件、导航栏和原生 tabBar 运行时配置。
- 影响 `packages/ui-tokens` 中小程序视觉 token 的默认值和模式化语义。
- 影响 `docs/miniapp-global-ui-design-prd.md`、`docs/miniapp-visual-illustration-system.md` 和 `openspec/specs/visual-system/spec.md` 的事实口径。
- 不影响 `apps/admin` 后台、不影响 `apps/api` 业务接口、不新增服务端数据表或远端主题同步。
