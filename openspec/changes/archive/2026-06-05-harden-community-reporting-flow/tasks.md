- [x] 3.1 扩展后台统一审核队列，支持按举报状态、原因、目标类型和优先级筛选社区举报。

- [x] 3.4 扩展举报处理接口，支持保留内容、隐藏内容、移除内容和标记误报，并拒绝重复处理状态已变化的举报。
- [x] 3.5 举报处理时分别记录举报处理结果、内容治理审计、作者治理审计和后台运营事件。

## 4. 通知与风险联动

- [x] 4.1 举报提交成功时返回受理反馈，但不默认创建持久化社区通知。
- [x] 4.2 举报人工处理完成后，为举报人创建克制的处理结果通知，不暴露后台备注、处罚细节或举报相关隐私。
- [x] 4.3 因举报处理隐藏或移除内容时，为内容作者创建可公开的内容处理通知，不暴露举报人身份或举报明细。
- [x] 4.4 将人工处理为隐藏或移除的举报计入被举报作者近期有效举报风险。
- [x] 4.5 确保待处理、保留内容和误报举报不影响被举报作者的低风险短文本快速通过资格。
