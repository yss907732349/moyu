## Why

现有小程序用户端已经形成暗黑忍者 RPG 视觉，但在真实办公场景中，用户需要一个更低暴露度的浏览入口，在不破坏主视觉基线的前提下查看核心摸鱼数据、日报和论坛内容。新增“摸鱼模式 / 办公表格伪装模式”可以把产品核心爽点伪装成工作表界面，提升上班场景下的可用性和记忆点。

## What Changes

- 新增小程序端 `stealth_workbench_mode` 功能入口，用户可从首页进入独立的摸鱼模式页面。
- 新增摸鱼模式工作台首页，以办公表格视觉展示今日已摸金额、距下班时间、当前工作状态、今日进度、日报入口和论坛入口。
- 新增表格版日报浏览体验，支持日报列表和日报详情，不展示插画、icon、RPG 面板或社区化装饰。
- 新增表格版论坛浏览体验，支持帖子列表、帖子详情、评论/回复只读查看、帖子点赞和帖子收藏。
- 摸鱼模式论坛第一版不支持发帖、评论、回复、举报、消息中心、我的帖子、收藏管理或图片上传。
- 摸鱼模式帖子图片默认不直接渲染，只显示附件数量和查看入口；用户可打开附件清单，再打开单张图片的无动画全屏查看层，点击任意位置立即关闭。
- 新增摸鱼模式专用办公表格 UI 规则，包括统一单元格、固定行列、细网格、无动画、无插画 icon、无头像徽章、无深色 RPG 背景和无游戏化按钮。
- 不改变现有首页、社区、日报、个人中心、补给铺、生存账本或底部 tabBar 的主视觉和业务能力。

## Capabilities

### New Capabilities

- `stealth-workbench-mode`: 定义小程序端摸鱼模式的入口、页面范围、办公表格视觉、核心数据展示、日报浏览、论坛轻互动和附件快速查看边界。

### Modified Capabilities

- `feature-registry`: 新增 `stealth_workbench_mode` 功能入口的公开展示、路由和非开放状态拦截要求。
- `visual-system`: 明确摸鱼模式是经 OpenSpec 定义的视觉例外，可豁免暗黑忍者 RPG 主视觉，但必须保持移动端可读性和业务边界。

## Impact

- 小程序端：新增摸鱼模式页面、表格版日报页面、表格版论坛列表和详情页面；首页新增入口或通过 feature registry 展示入口。
- 小程序服务：复用现有工作档案、首页工作价值计算、日报客户端服务和社区客户端服务；不新增独立后端 API。
- 共享契约：可能新增 `stealth_workbench_mode` feature key 或本地默认入口配置；不改变日报、社区、工作档案接口契约。
- 视觉文档与规格：补充摸鱼模式作为独立浅色办公表格视觉系统的规则，避免误用现有 RPG 组件。
- 验证：需要补充小程序端摸鱼模式静态/结构检查，覆盖无插画 icon、固定表格规则、论坛能力边界和附件查看层规则。
