## 2. API 评论回复审核分流

- [x] 2.1 梳理现有社区评论和回复创建流程，定位低成本审核、用户治理状态、频率限制和重复内容判断入口。
- [x] 2.2 实现统一评论/回复审核决策函数，组合文本风险、用户风险、频率限制、重复内容和治理状态。
- [x] 2.3 对可信低风险短评论落地 `auto_approve`，创建或更新为 `approved` 状态并记录决策摘要。
- [x] 2.4 对可信低风险短回复落地 `auto_approve`，创建或更新为 `approved` 状态并记录决策摘要。
- [x] 2.5 对明确违规评论和回复落地 `auto_reject`，设为 `rejected` 或拒绝公开，并保留后台可见原因。
- [x] 2.6 对新用户、近期违规用户、被有效举报用户、`limited` 用户、频率异常和灰区内容落地 `manual_review`。
- [x] 2.7 确保 `muted` 和 `banned` 用户仍被写入口拦截，不进入自动分流。
- [x] 2.8 确保帖子创建流程不因本 change 自动公开，低风险帖子仍保持 `pending`。
