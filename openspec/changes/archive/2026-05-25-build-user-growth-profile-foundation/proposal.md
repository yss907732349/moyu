## Why

首页工作价值闭环已经具备工作档案、实时已摸金额、工作状态、发薪/休息/节假日倒计时和隐藏模式，但用户身份与成长承接仍停留在个人中心静态演示。下一步需要建立真实的“隐者”资料基础，让阵营、等级、隐币、签到、称号、社区身份和后续日报能力有稳定依赖。

## What Changes

- 新增微信小程序登录第一版，用于创建或识别应用内用户，并为 `/me/*` 语义提供真实本人上下文。
- 新增“我的页面 / 用户成长资料”基础契约，覆盖职业类型、阵营、默认昵称、默认头像 key、等级、经验、隐币、能量、连续签到、称号和徽章展示占位。
- 首次打开“我的”且未完成成长资料时，展示创建隐者档案流程；用户必须选择职业类型，系统据此分配阵营并初始化成长资料。
- 新增每日签到第一版，同一用户同一自然日只能签到一次，成功后发放配置化的经验、隐币或能量奖励并更新连续签到。
- 将个人中心功能宫格接入 `profile_feature_grid` 的 feature registry 配置，替换页面内静态功能入口。
- 继续保留明确标记的本地开发临时本人上下文作为开发 fallback，但生产路径不应再把成长资料绑定到固定占位用户。
- 称号、徽章和默认头像仅做展示占位、字段预留和资源 key 预留，不做商店、购买、复杂解锁或真实头像上传。
- 不包含手机号授权、完整 session 刷新、多设备管理、账号注销、社交关系、社区发帖身份约束或后台用户运营能力。

## Capabilities

### New Capabilities

- `user-growth-profile`: 用户微信登录、首次创建隐者档案、职业到阵营分配、成长资源、每日签到和我的页资料展示。

### Modified Capabilities

- `feature-registry`: 个人中心功能宫格应通过 `profile_feature_grid` 获取公开入口配置，并按入口状态控制导航和阻断提示。

## Impact

- `packages/shared`: 新增用户成长资料、职业类型、阵营、签到、微信登录响应和资料 API 响应契约；必要时修正当前阵营 key 与产品命名的一致性。
- `apps/api`: 新增微信登录接口、用户/成长资料持久化模型、本人资料接口、签到接口，并将本人上下文从开发占位迁移到登录态优先。
- `apps/miniapp`: 新增登录与用户资料客户端服务、本地资料快照缓存、首次创建隐者档案流程、个人中心真实数据渲染、签到交互和 feature registry 宫格接入。
- `apps/api/prisma`: 新增用户、微信身份、成长资料和签到记录相关模型或等价持久化结构。
- `scripts`: 新增或扩展验证脚本，覆盖共享契约、API 契约、个人中心入口、签到幂等和 feature registry 宫格消费。
