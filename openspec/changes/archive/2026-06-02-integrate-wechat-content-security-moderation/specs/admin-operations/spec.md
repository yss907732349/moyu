## ADDED Requirements

### Requirement: 后台内容安全摘要展示

后台运营中心 SHALL 在统一审核队列、审核详情和工作台摘要中展示归一化内容安全结果，帮助管理员区分本地规则、微信内容安全、mock、供应商错误和人工复核来源。

#### Scenario: 队列展示内容安全摘要

- **WHEN** 后台用户读取统一审核队列
- **THEN** 每个相关队列项 SHALL 展示内容安全来源、归一化风险标签、简短原因、审核决策、用户风险原因和人工复核原因
- **AND** 队列项 SHALL 能区分本地规则、微信文本审核、微信图片审核、mock 和供应商降级

#### Scenario: 工作台统计内容安全分流

- **WHEN** 后台用户读取运营中心工作台
- **THEN** API SHALL 返回自动通过、自动驳回、微信待回调、供应商无法确认和人工复核的统计摘要
- **AND** 摘要 SHALL 支持社区、日报评论和大陆新闻评论的来源区分

#### Scenario: 审核详情展示图片审核状态

- **WHEN** 后台用户查看带图社区帖子的审核详情
- **THEN** API SHALL 返回每张图片的归一化审核状态、供应商来源、脱敏任务标识和可理解原因
- **AND** 后台 SHALL 可以据此人工通过、驳回或隐藏帖子

### Requirement: 后台内容安全敏感字段边界

后台运营中心 SHALL 过滤微信内容安全和供应商审核的敏感字段，只返回后台处理所需的归一化摘要。

#### Scenario: 队列响应不暴露微信敏感字段

- **WHEN** 后台读取工作台、审核队列或审核详情
- **THEN** 响应 SHALL NOT 包含微信 `secret`、`access_token`、`sessionKey`、完整 openid、完整 unionid、完整供应商原始响应或服务端环境变量
- **AND** 如需展示任务标识，响应 SHALL 使用脱敏 `trace_id` 或不可逆摘要

#### Scenario: 后台筛选风险不依赖原始响应

- **WHEN** 后台用户按内容安全来源、风险标签或人工复核原因筛选队列
- **THEN** API SHALL 基于归一化字段完成筛选
- **AND** API SHALL NOT 需要返回供应商完整原始响应来支撑筛选
