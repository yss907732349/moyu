## Why

当前后台日报运营界面仍以整期日报为主工作单元，`今日参悟`、`大陆新闻`、`离谱卷宗` 只是同一编辑表单下的切换页签。运营人员无法明确地只保存、只校验、只预览或只发布一个板块，容易在更新单个板块时误触整期保存和整期发布流程。

这次变更要把三个日报板块拆成独立运营工作台，使每个板块都可以独立发文，同时保留整期日报的日期、首页摘要和发布汇总能力。

## What Changes

- 后台日报运营界面将从“整期编辑 + 板块 tab”重构为三个独立板块工作区：`今日参悟`、`大陆新闻`、`离谱卷宗`。
- 每个板块工作区提供独立的编辑、保存、校验、预览和发布入口。
- 单板块保存只更新目标板块内容，不覆盖其他板块的草稿内容。
- 单板块发布只发布目标板块内容，不要求其他板块同时达到发布条件。
- 发布检查支持按板块展示阻断项和提示项，避免其他板块的问题干扰当前板块发布判断。
- 整期级操作继续保留，用于维护日报日期、总标题、首页摘要、定时发布、整期发布和归档，但不得作为单板块发文的必经路径。
- 不改变小程序公开侧日报结构：公开侧仍看到同一业务日期下的今日参悟、栏目入口和文章详情。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `daily-content-feed`: 调整日报后台编辑与发布契约，要求支持按板块保存、校验、预览和发布。
- `admin-operations`: 调整后台运营中心日报运营体验，要求三个日报板块在 UI 和操作语义上完全独立。

## Impact

- `apps/admin`: 重构日报运营页面布局、状态管理、按钮动作、校验展示和预览展示。
- `apps/api`: 增加或调整后台日报单板块保存、校验、预览和发布接口；现有整期接口继续兼容。
- `packages/shared`: 如需复用契约，补充单板块编辑请求、校验结果或板块级预览类型。
- `openspec/specs/daily-content-feed`: 增加板块级后台编辑与发布要求。
- `openspec/specs/admin-operations`: 增加日报运营界面拆分要求。
- 验证命令重点覆盖 `pnpm verify:daily-content-feed-api`、`pnpm verify:admin-daily-content-feed`、`pnpm typecheck`。
