## Why

`摸鱼侠` 的 MVP 已明确需要轻社区能力，但当前社区页仍是视觉演示数据，缺少帖子、评论、互动、举报、审核和隐私边界的正式契约。现在需要先定义 `community-lite` 第一阶段范围，避免后续实现时把未审核内容、敏感工作档案或生存账单数据误接入公开社区。

## What Changes

- 新增轻社区第一阶段契约，覆盖帖子模型、分区、公开身份、互动、评论、举报、审核状态和公开展示规则。
- 定义社区分区：推荐、键影、运影、策影、行影、魔王吐槽。
- 定义阵营专区发帖规则：阵营专区只允许同阵营用户发帖，所有登录用户可以浏览、点赞、收藏、评论和举报；推荐与魔王吐槽不限制发帖阵营。
- 定义未登录或未创建隐者档案时的社区访问边界：用户只能浏览推荐分区的帖子列表预览；进入帖子详情、切换到其他分区、发帖、评论、点赞、收藏或举报时，应跳转到“我的”页引导登录并创建阵营身份。
- 定义内容审核生命周期：帖子和评论提交后进入 `pending`，只有人工审核通过成为 `approved` 后才能公开展示；`rejected` 和 `hidden` 不公开。
- 定义第一阶段 API 边界：帖子列表、详情、发帖、评论、点赞、收藏和举报。
- 定义小程序页面边界：社区首页、分区切换、发帖页和帖子详情页。
- 明确隐私边界：社区公开内容只能展示用户成长资料中的公开身份信息，不展示薪资、工作时间、工作档案、生存账单、消费统计、CPS 平台来源或微信身份标识。

## Capabilities

### New Capabilities

- `community-lite`: 轻社区帖子、分区、评论、互动、举报、审核生命周期、公开身份展示和隐私边界。

### Modified Capabilities

- 无。

## Impact

- 后续将影响 `packages/shared` 的社区枚举、类型、响应契约和校验函数。
- 后续将影响 `apps/api` 的社区模块、本人上下文接入、Prisma 数据模型、帖子/评论/互动/举报接口和最小审核接口。
- 后续将影响 `apps/miniapp` 的社区首页真实数据接入、未登录推荐预览、登录/建档引导跳转、发帖页、帖子详情页、互动状态和错误/审核提示。
- 后续将影响 `apps/admin` 或管理 API 的最小人工审核入口，但本 change 只定义契约，不实现完整运营后台。
